# Lec 3: Learn STL and Java Collections
# Topic: Java STL (Java Standard Library for DSA)

Unlike C++, Java does **not** have an official library called **STL (Standard Template Library)**. The equivalent of STL in Java is the **Java Collections Framework (JCF)** along with utility classes such as `Arrays`, `Collections`, `Math`, and commonly used classes from `java.util`.

> **C++ STL = Java Collections Framework (JCF) + Utility Classes**

---

# What is Java STL?

Java STL refers to the collection of **pre-built data structures**, **algorithms**, and **utility classes** provided by Java to simplify programming.

Instead of implementing data structures manually, Java provides optimized implementations like:

- ArrayList
- LinkedList
- Stack
- Queue
- Deque
- PriorityQueue
- HashSet
- TreeSet
- HashMap
- TreeMap

These are heavily used in DSA and Competitive Programming.

---

# Why Learn Java STL?

Without Java STL:

- Need to implement data structures manually.
- More code.
- Higher chance of bugs.

With Java STL:

- Less code
- Optimized performance
- Easy to use
- Faster problem solving

Example

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
```

Creating a Min Heap takes only one line.

# List

Stores ordered elements.

Characteristics

- Ordered
- Duplicate elements allowed
- Index based

Most Used

```java
ArrayList<Integer> list = new ArrayList<>();
```

Operations

```java
list.add(10);

list.get(0);

list.remove(0);

list.set(0,100);

list.size();
```

Time Complexity

| Operation | Complexity |
|------------|------------|
| add() | O(1)* |
| get() | O(1) |
| remove(index) | O(n) |

\* Amortized.

# Comparable vs Comparator

### Comparable

Natural ordering.

```java
Collections.sort(list);
```

### Comparator

Custom ordering.

```java
Collections.sort(list,(a,b)->b-a);
```

Useful for:

- Descending order
- Object sorting
- Multi-key sorting

# Common Mistakes

### Using `==` for String Comparison

Wrong

```java
s1 == s2
```

Correct

```java
s1.equals(s2)
```

---

### Using `Stack`

Prefer

```java
ArrayDeque
```

for better performance.

---

### Removing Integer from ArrayList

Wrong

```java
list.remove(1);
```

Removes index `1`.

Correct

```java
list.remove(Integer.valueOf(1));
```

# Competitive Programming Tips

- Learn `HashMap`, `HashSet`, `PriorityQueue`, and `ArrayDeque` thoroughly—they appear in a large number of DSA problems.
- Memorize the time complexities of all major collections.
- Use `Arrays.sort()` for arrays and `Collections.sort()` for lists.
- Prefer `HashMap` unless sorted keys are required; use `TreeMap` only when ordering matters.
- Use `StringBuilder` instead of repeated string concatenation inside loops.

---

# Revision Notes

- Java STL is primarily the Java Collections Framework plus utility classes.
- `ArrayList` is the Java equivalent of C++ `vector`.
- `HashMap` and `HashSet` provide average O(1) operations.
- `PriorityQueue` is a Min Heap by default.
- `TreeMap` and `TreeSet` maintain sorted order.
- `ArrayDeque` is preferred over `Stack`.
- `Arrays` and `Collections` contain many useful built-in algorithms.
