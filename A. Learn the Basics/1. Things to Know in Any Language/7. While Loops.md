# Lec 1: Things to Know in C++ / Java / Python
# Topic: While Loops

A **while loop** is an **iteration (looping) statement** that repeatedly executes a block of code **as long as a given condition is true**.

Unlike a `for` loop, a `while` loop is generally used when the **number of iterations is not known in advance**.

---

# Why Do We Need While Loops?

Suppose you want to keep asking a user for input until they enter `0`.

Since you don't know how many times the user will enter a value, a `while` loop is the perfect choice.

```
Input → Check Condition → Execute → Repeat
```

Examples:

- Read input until EOF
- Process linked lists
- BFS/DFS traversal
- Binary Search
- Reverse a Number
- Extract Digits

---

# What is a While Loop?

A `while` loop checks the condition **before** executing the loop body.

If the condition is false initially, the loop **does not execute even once**.

---

# Syntax

```java
while(condition){
    // Code
}
```

---

# Flow of Execution

```
         Start
           │
           ▼
    Check Condition
      /         \
   True        False
     │            │
     ▼            ▼
 Execute Body     End
     │
     ▼
 Update Variable
     │
     └────────────►
```

# Infinite While Loop

```java
while(true){

    System.out.println("Hello");
}
```

The loop never ends unless a `break` statement is used.

---

# Using `break`

Stops the loop immediately.

```java
int i = 1;

while(i <= 10){

    if(i == 5){
        break;
    }

    System.out.println(i);

    i++;
}
```

Output

```
1
2
3
4
```

---

# Using `continue`

Skips the current iteration.

```java
int i = 0;

while(i < 5){

    i++;

    if(i == 3){
        continue;
    }

    System.out.println(i);
}
```

Output

```
1
2
4
5
```

Notice that `i++` is performed **before** `continue` to avoid an infinite loop.

# While vs For Loop

| Feature | While Loop | For Loop |
|----------|------------|----------|
| Iterations Known? | Usually No | Usually Yes |
| Initialization | Outside Loop | Inside Loop |
| Update | Inside Loop | In Loop Header |
| Best For | Unknown repetitions | Fixed repetitions |

---

# When Should You Use While?

Use `while` when:

- Number of iterations is unknown.
- Reading input until a condition changes.
- Traversing linked lists.
- Binary Search.
- Tree traversal.
- Graph traversal (BFS/DFS).
- Processing digits of a number.

Example

```java
while(n > 0){

    int digit = n % 10;

    n /= 10;
}
```

# Common Mistakes

## 1. Forgetting to Update the Variable

Wrong

```java
int i = 1;

while(i <= 5){
    System.out.println(i);
}
```

Output

```
Infinite Loop
```

Correct

```java
i++;
```

---

## 2. Wrong Condition

Wrong

```java
while(i >= 5)
```

when `i = 1`.

The loop never executes.

---

## 3. Updating After `continue`

Wrong

```java
while(i < 5){

    if(i == 2)
        continue;

    i++;
}
```

If `i == 2`, `continue` skips `i++`, causing an infinite loop.

Correct

```java
i++;

if(i == 2)
    continue;
```

---

## 4. Changing Loop Variable Incorrectly

```java
while(i < n){
    i += 2;
}
```

This skips alternate values.

Use only if intended.

---

# Competitive Programming Tips

- Use `while` for problems where the number of iterations is unknown.
- Most digit-based problems (`Reverse Number`, `Palindrome`, `Count Digits`) use `while(n > 0)`.
- Always ensure the loop variable is updated.
- Use `break` for early termination.
- Be careful when using `continue` to avoid infinite loops.

---

# Revision Notes

- A `while` loop executes as long as the condition is true.
- The condition is checked **before** each iteration.
- It may execute **zero or more times**.
- Initialization is done outside the loop.
- Updating the loop variable is your responsibility.
- Use `while` when the number of iterations is unknown.
- `break` exits the loop immediately.
- `continue` skips the current iteration.
- Forgetting to update the loop variable causes an infinite loop.
- `while(n > 0)` is a common pattern in number-based DSA problems.
