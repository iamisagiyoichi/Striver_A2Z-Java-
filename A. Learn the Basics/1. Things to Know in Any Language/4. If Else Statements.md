# Lec 1: Things to Know in C++ / Java / Python
# Topic: If-Else Statements (Decision Making)

An **If-Else Statement** is a **conditional statement** that allows a program to make decisions based on whether a condition is **true** or **false**.

Instead of executing every statement sequentially, the program can choose which block of code to execute.

---

# Why Do We Need If-Else?

In real life, we make decisions every day.

Examples:

- If it is raining → Take an umbrella.
- If marks are above 90 → Grade A.
- If age is 18 or above → Eligible to vote.

Programming follows the same concept.

```
Condition
     │
     ▼
  Is it True?
   /       \
 Yes       No
 │          │
Execute    Execute
If Block   Else Block
```

---

# Syntax of `if`

```java
if(condition){
    // Code executes if condition is true
}
```

Example

```java
int age = 20;

if(age >= 18){
    System.out.println("Eligible to Vote");
}
```

Output

```
Eligible to Vote
```

---

# Flow of Execution

```text
Start
  │
Check Condition
  │
 ┌───────┐
 │ True? │
 └───────┘
  │   │
Yes   No
 │     │
Execute Skip
 │     │
 └──► End
```

# `if-else` Statement

Used when there are **two possible outcomes**.

### Syntax

```java
if(condition){
    // True Block
}
else{
    // False Block
}
```

Example

```java
int num = 15;

if(num % 2 == 0){
    System.out.println("Even");
}
else{
    System.out.println("Odd");
}
```

Input

```
15
```

Output

```
Odd
```

---

# `if-else if-else`

Used when there are **multiple conditions**.

### Syntax

```java
if(condition1){

}
else if(condition2){

}
else if(condition3){

}
else{

}
```

Example

```java
int marks = 82;

if(marks >= 90){
    System.out.println("Grade A");
}
else if(marks >= 75){
    System.out.println("Grade B");
}
else if(marks >= 60){
    System.out.println("Grade C");
}
else{
    System.out.println("Fail");
}
```

Output

```
Grade B
```

---

# Nested If

An `if` statement inside another `if` statement.

```java
int age = 20;
boolean hasLicense = true;

if(age >= 18){
    if(hasLicense){
        System.out.println("Can Drive");
    }
}
```

Output

```
Can Drive
```

# Logical Operators in Conditions

## AND (`&&`)

Returns `true` only if **both conditions** are true.

```java
if(age >= 18 && hasLicense){
    System.out.println("Allowed");
}
```

---

## OR (`||`)

Returns `true` if **at least one condition** is true.

```java
if(age >= 18 || hasPermission){
    System.out.println("Allowed");
}
```

---

## NOT (`!`)

Reverses the boolean value.

```java
boolean rain = false;

if(!rain){
    System.out.println("Go Outside");
}
```

# Boolean Expressions

Every condition in an `if` statement must evaluate to a boolean value.

```java
boolean isPrime = true;

if(isPrime){
    System.out.println("Prime");
}
```

---

# Order of Evaluation

Conditions are checked **from top to bottom**.

Once a condition becomes **true**, the remaining `else if` conditions are skipped.

Example

```java
int x = 95;

if(x >= 90){
    System.out.println("A");
}
else if(x >= 80){
    System.out.println("B");
}
```

Output

```
A
```

Even though `95 >= 80` is also true, the second condition is never checked.

---

# Short-Circuit Evaluation

Java uses **short-circuit evaluation** for `&&` and `||`.

### AND (`&&`)

If the first condition is `false`, the second condition is **not evaluated**.

Example

```java
if(a != 0 && b / a > 2){
    ...
}
```

If `a == 0`, Java skips `b / a` and avoids a division-by-zero error.

---

### OR (`||`)

If the first condition is `true`, the second condition is **not evaluated**.

Example

```java
if(isAdmin || hasPermission){
    ...
}
```

If `isAdmin` is `true`, Java doesn't check `hasPermission`.

---

# Ternary Operator

A shorthand for simple `if-else`.

### Syntax

```java
condition ? valueIfTrue : valueIfFalse;
```

Example

```java
int num = 5;

String ans = (num % 2 == 0) ? "Even" : "Odd";

System.out.println(ans);
```

Output

```
Odd
```

# Real-Life Examples

### Find Maximum of Two Numbers

```java
int a = 20;
int b = 15;

if(a > b){
    System.out.println(a);
}
else{
```

### Check Positive, Negative, or Zero

```java

    System.out.println("Positive");
}
else if(x < 0){
    System.out.println("Negative");
}
else{
    System.out.println("Zero");
```

### Check Leap Year

```java
if((year % 400 == 0) ||
   (year % 4 == 0 && year % 100 != 0)){
    System.out.println("Leap Year");
}
else{
    System.out.println("Not Leap Year");
}
```

# Common Mistakes

## 1. Using `=` Instead of `==`

Wrong

```java
if(a = 5)
```

Correct

```java
if(a == 5)
```

---

## 2. Wrong Order of Conditions

Wrong

```java
if(marks >= 60){
    System.out.println("Pass");
}
else if(marks >= 90){
    System.out.println("Grade A");
}
```

`marks >= 90` will never be reached.

Correct

```java
if(marks >= 90){
    ...
}
else if(marks >= 60){
    ...
}
```

---

## 3. Missing Braces

Wrong

```java
if(a > b)
    System.out.println(a);
    System.out.println(b);
```

Only the first statement belongs to the `if`.

Correct

```java
if(a > b){
    System.out.println(a);
    System.out.println(b);
}
```

---

## 4. Comparing Strings with `==`

Wrong

```java
if(s1 == s2)
```

Correct

```java
if(s1.equals(s2))
```

---

## 5. Confusing `&&` and `||`

```java
&&  → Both conditions must be true

||  → At least one condition must be true
```

---

# Competitive Programming Tips

- Always check **edge cases** in conditions.
- Place the **most restrictive or highest-priority condition first**.
- Use `&&` and `||` carefully to avoid logical errors.
- Avoid deeply nested `if` statements when possible; simplify the logic.
- Use the ternary operator only for short, simple conditions.

---

# Revision Notes

- `if` executes code only when the condition is true.
- `if-else` provides two execution paths.
- `else if` is used for multiple conditions.
- Nested `if` allows decisions inside decisions.
- Conditions always return a boolean value.
- `&&` means **AND**, `||` means **OR**, `!` means **NOT**.
- Java uses **short-circuit evaluation** for logical operators.
- Ternary operator is a compact form of `if-else`.
- Use `==` for primitive comparisons and `.equals()` for string comparisons.
- Keep conditions simple and readable.
