# Lec 1: Things to Know in C++ / Java / Python
# Topic: Functions (Pass by Value & Pass by Reference)

A **Function** (also called a **Method** in Java) is a reusable block of code that performs a specific task.

Instead of writing the same code multiple times, we write it once inside a function and call it whenever needed.

---

# Why Do We Need Functions?

Suppose you want to calculate the sum of two numbers multiple times.

Without functions:

```java
int sum = a + b;
System.out.println(sum);

sum = x + y;
System.out.println(sum);

sum = p + q;
System.out.println(sum);
```

With functions:

```java
add(a, b);
add(x, y);
add(p, q);
```

Functions make code:

- Reusable
- Modular
- Easy to debug
- Easy to maintain
- More readable

---

# What is a Function?

A function takes **input (parameters)**, performs some **processing**, and may **return an output**.

```
Input
   │
   ▼
Function
   │
Processing
   │
   ▼
Output
```

Example

```java
add(5, 10)
```

```
Input → 5, 10

Processing → 5 + 10

Output → 15
```

---

# Syntax of a Function (Java)

```java
returnType functionName(parameters){

    // Code

    return value;
}
```

Example

```java
static int add(int a, int b){

    return a + b;
}
```

Calling the function

```java
int ans = add(5, 7);

System.out.println(ans);
```

Output

```
12
```

# Function Call


1. Control goes to `add()`.

3. Function executes.
4. Result is returned.
5. Control comes back.

# Local Variables

Variables declared inside a function exist only inside that function.

```java

    int x = 10;
}
```

Outside the function:

```java
System.out.println(x);
```

Compilation Error.

# Parameter vs Argument

### Parameter

Variables declared in the function definition.

```java
static void add(int a, int b)
```

`a` and `b` are parameters.

---

### Argument

Values passed while calling the function.

```java
add(5, 7);
```

`5` and `7` are arguments.

---

# Function Overloading

Multiple functions with the same name but different parameter lists.

```java
static int add(int a, int b){

    return a + b;
}

static double add(double a, double b){

    return a + b;
}
```

Java decides which function to call based on the arguments.

# Pass by Value

## What is Pass by Value?

In **Pass by Value**, a **copy** of the variable is passed to the function.

Changes made inside the function **do not affect** the original variable.

### Java always uses Pass by Value.

---

### Example

```java
static void change(int x){

    x = 100;
}

public static void main(String[] args){

    int a = 10;

    change(a);

    System.out.println(a);
}
```

Output

```
10
```

Explanation

```
a = 10

↓

Copy created

↓

x = 10

↓

x becomes 100

↓

Original a remains 10
```

Memory

```
Main

a = 10

↓

Function

x = 10

↓

x = 100

↓

Function Ends

a = 10
```

---

# Pass by Reference

## What is Pass by Reference?

In pass by reference, the **address (reference)** of the variable is passed.

Changes inside the function affect the original data.

Languages like **C++** support true pass-by-reference.

Example (C++)

```cpp
void change(int &x){

    x = 100;
}
```

Now

```cpp
int a = 10;

change(a);
```

Output

```
100
```

---

# How Java Handles Objects

Java does **not** support true pass-by-reference.

However, when passing objects or arrays, Java passes a **copy of the reference**.

This means the function can modify the object's contents, but it cannot make the caller reference a different object.

---

## Example with Array

```java
static void change(int[] arr){

    arr[0] = 100;
}

public static void main(String[] args){

    int[] arr = {10,20,30};

    change(arr);

    System.out.println(arr[0]);
}
```

Output

```
100
```

Why?

```
Copy of Reference

↓

Both references point to

[10,20,30]

↓

Modify first element

↓

Original array changes
```

---

# Example with String

```java
static void change(String s){

    s = "World";
}

public static void main(String[] args){

    String s = "Hello";

    change(s);

    System.out.println(s);
}
```

Output

```
Hello
```

Reason

Strings are immutable.

The function only changes its local copy of the reference.

# Recursive Functions

A function can call itself.

Example

```java
static void print(int n){

    if(n == 0)
        return;

    System.out.println(n);

    print(n - 1);
}
```

Output

```
5
4
3
2
1
```

Recursion is discussed in detail later in the DSA sheet.

# Common Mistakes

## 1. Forgetting to Return

Wrong

```java
static int add(int a, int b){

    a + b;
}
```

Correct

```java
return a + b;
```

---

## 2. Returning Wrong Type

Wrong

```java
static int fun(){

    return "Hello";
}
```

Compilation Error.

---

## 3. Assuming Java Uses Pass by Reference

Wrong

```java
change(a);
```

Expecting `a` to change.

Primitive variables never change outside the function.

---

## 4. Modifying Strings

Wrong

```java
void change(String s){

    s = "ABC";
}
```

Original string remains unchanged.

---

# Competitive Programming Tips

- Divide large problems into smaller functions.
- Keep functions short and reusable.
- Pass arrays directly; changes will be visible because the object is shared.
- Use `long` as the return type when answers may exceed the `int` range.
- Avoid unnecessary recursion when iteration is simpler and more efficient.

---

# Revision Notes

- Functions improve code reusability and readability.
- A function can take parameters and return a value.
- `void` functions do not return anything.
- Parameters receive values; arguments supply values.
- Java supports function overloading.
- Java is **always pass by value**.
- Primitive variables are not modified outside the function.
- Arrays and objects can have their contents modified because a copy of their reference is passed.
- Strings are immutable.
- Recursion is a function calling itself.
