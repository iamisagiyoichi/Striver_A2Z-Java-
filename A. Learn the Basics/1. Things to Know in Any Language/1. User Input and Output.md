# Lec 1: Things to Know in C++ / Java / Python
# Topic: User Input / Output (I/O)

User Input/Output (I/O) is one of the most fundamental concepts in programming. Every program takes some **input**, processes it using an algorithm, and produces the desired **output**.

```
Input  →  Processing  →  Output
```

Example:

```
Input:
5 10

Processing:
Add both numbers

Output:
15
```

---

# Why is Input/Output Important?

- Every competitive programming problem requires reading input and printing output.
- Efficient input methods are crucial when handling large datasets.
- Choosing the right I/O method can significantly reduce execution time.

---

# Input and Output in Java

## 1. Scanner Class (Beginner-Friendly)

The `Scanner` class is the easiest way to read input from the user.

### Import

```java
import java.util.Scanner;
```

### Creating a Scanner Object

```java
Scanner sc = new Scanner(System.in);
```

- `System.in` refers to the keyboard input stream.
- `sc` is the object used to read data.

---

# Difference Between `next()` and `nextLine()`

| Method | Reads |
|---------|------|
| `next()` | One word (stops at space) |
| `nextLine()` | Entire line including spaces |

Example

Input

```
Hello World
```

```java
String a = sc.next();
```

Output

```
Hello
```

```java
String b = sc.nextLine();
```

Output

```
Hello World
```

# Output in Java

## println()

Prints the value and moves to the next line.

```java
System.out.println("Hello");
```

Output

```
Hello
```

---

## print()

Prints without moving to the next line.

```java
System.out.print(10);
System.out.print(20);
```

Output

```
1020
```

---

## printf()

Used for formatted output.

```java
double pi = 3.141592;

System.out.printf("%.2f", pi);
```

Output

```
3.14
```

# Complete Example

```java
import java.util.*;

public class Main {

    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

        int a = sc.nextInt();
        int b = sc.nextInt();

        System.out.println(a + b);
    }
}
```

Input

```
10 20
```

Output

```

```

# Fast Input for Competitive Programming

`Scanner` is easy to use but relatively slow for large inputs.

For competitive programming, prefer:

- `BufferedReader`
- `FastScanner` (custom implementation)

---

## BufferedReader

### Import

```java
import java.io.*;
```

### Example

```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

int n = Integer.parseInt(br.readLine());

System.out.println(n);
```

---

### Reading Multiple Integers

```java
String[] input = br.readLine().split(" ");

int a = Integer.parseInt(input[0]);
int b = Integer.parseInt(input[1]);
```

Input

```
5 10
```

---

# BufferedReader vs Scanner

| Feature | Scanner | BufferedReader |
|----------|----------|---------------|
| Easy to Use | ✅ | ❌ |
| Speed | Slow | Fast |
| Competitive Programming | Not Preferred | Preferred |
| Parsing Needed | No | Yes |

---

# Why is BufferedReader Faster?

`Scanner`

- Uses regular expressions internally.
- Performs additional parsing and validation.
- More convenient but slower.

`BufferedReader`

- Reads an entire line at once.
- Does not perform automatic parsing.
- Less overhead, making it significantly faster.

---

# Output Using BufferedWriter

```java
BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));

bw.write("Hello");
bw.newLine();

bw.flush();
```

# Common Input Mistake

## Mixing `nextInt()` and `nextLine()`

Wrong

```java
int n = sc.nextInt();

String s = sc.nextLine();
```

Input

```
5
Hello
```

Output

```
"" (empty string)
```

Reason

`nextInt()` leaves the newline (`\n`) character in the input buffer. The following `nextLine()` immediately reads this leftover newline.

---

## Correct Way

```java
int n = sc.nextInt();
sc.nextLine();

String s = sc.nextLine();
```

# Competitive Programming Tips

- Use `BufferedReader` or a custom `FastScanner` for faster input.
- Use `PrintWriter` or `BufferedWriter` for efficient output.
- Read arrays using loops.
- Use `long` when constraints involve values up to `10^9` or multiplications that may overflow `int`.
- Always check the input format before writing your code.

---

# Revision Notes

- **Input → Process → Output** is the basic flow of every program.
- `Scanner` is beginner-friendly but slower.
- `BufferedReader` is preferred for competitive programming.
- `next()` reads one word; `nextLine()` reads an entire line.
- Use `charAt(0)` to read a character with `Scanner`.
- Read arrays using loops.
- `println()` prints with a newline, while `print()` does not.
- `printf()` is used for formatted output.
- Be careful when mixing `nextInt()` and `nextLine()`.
- Remember to `flush()` when using `BufferedWriter`.
