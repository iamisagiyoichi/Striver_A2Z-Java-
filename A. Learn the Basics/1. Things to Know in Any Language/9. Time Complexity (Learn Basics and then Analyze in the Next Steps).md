# Topic: Time Complexity (Basics)

Time Complexity is one of the most important concepts in **Data Structures and Algorithms (DSA)**. It measures **how the running time of an algorithm grows as the input size (`N`) increases**.

> **Time Complexity does NOT measure actual execution time (seconds).**
>
> It measures **how the number of operations increases with input size**.

---

# Why Do We Need Time Complexity?

Suppose there are two algorithms to find the maximum element.

### Algorithm 1

```java
for(int i = 0; i < n; i++){
    // Check every element
}
```

### Algorithm 2

```java
return arr[0];
```

If

```
N = 10
```

Algorithm 1 performs about **10 operations**.

If

```
N = 1,000,000
```

Algorithm 1 performs about **1,000,000 operations**.

Algorithm 2 always performs **1 operation**.

Hence,

```
Algorithm 2 is more efficient.
```

---

# What is Input Size (N)?

`N` represents the size of the input.

Examples

- Number of elements in an array
- Length of a string
- Number of nodes in a tree
- Number of vertices in a graph

Example

```java
int[] arr = new int[100];
```

Here,

```
N = 100
```

---

# Why Don't We Measure Time in Seconds?

Execution time depends on:

- CPU Speed
- Programming Language
- Compiler Optimizations
- Operating System
- Hardware

Example

```
Same Algorithm

Laptop A → 0.5 sec

Laptop B → 0.2 sec
```

But the **Time Complexity** remains the same.

---

# Big-O Notation

Big-O Notation represents the **worst-case growth** of an algorithm.

Example

```
O(1)
O(log N)
O(N)
O(N log N)
O(N²)
```

It tells us how fast the running time grows as `N` becomes very large.

---

# Common Time Complexities

| Complexity | Name | Performance |
|------------|------|-------------|
| O(1) | Constant | ⭐⭐⭐⭐⭐ |
| O(log N) | Logarithmic | ⭐⭐⭐⭐⭐ |
| O(√N) | Square Root | ⭐⭐⭐⭐ |
| O(N) | Linear | ⭐⭐⭐⭐ |
| O(N log N) | Linearithmic | ⭐⭐⭐ |
| O(N²) | Quadratic | ⭐⭐ |
| O(N³) | Cubic | ⭐ |
| O(2ᴺ) | Exponential | ❌ |
| O(N!) | Factorial | ❌ |

---

# 1. O(1) — Constant Time

The number of operations **does not depend on `N`**.

Example

```java
int x = arr[5];
```

No matter how large the array is, only one element is accessed.

Examples

- Access array element
- Assign variable
- Swap two variables

---

# 2. O(log N) — Logarithmic Time

The problem size is reduced by half in every iteration.

Example

```java
while(n > 1){
    n /= 2;
}
```

Iterations

```
16

↓

8

↓

4

↓

2

↓

1
```

Only **4 iterations**.

Examples

- Binary Search
- Fast Exponentiation

---

# 3. O(√N)

The loop runs only up to the square root of `N`.

Example

```java
for(int i = 1; i * i <= n; i++){

}
```

Examples

- Prime Checking
- Divisor Problems

---

# 4. O(N) — Linear Time

The number of operations increases directly with `N`.

Example

```java
for(int i = 0; i < n; i++){

}
```

If

```
N = 100
```

100 iterations.

Examples

- Traversing an array
- Counting frequency
- Finding maximum

---

# 5. O(N log N)

Usually appears in efficient sorting algorithms.

Examples

- Merge Sort
- Heap Sort
- Quick Sort (Average Case)

---

# 6. O(N²) — Quadratic Time

Two nested loops.

Example

```java
for(int i = 0; i < n; i++){

    for(int j = 0; j < n; j++){

    }

}
```

Iterations

```
N × N
```

Examples

- Bubble Sort
- Selection Sort
- Matrix Comparison

---

# 7. O(N³)

Three nested loops.

```java
for(...){

    for(...){

        for(...){

        }

    }

}
```

Examples

- Floyd Warshall
- Naive Matrix Multiplication

---

# 8. O(2ᴺ)

Every recursive call branches into two more calls.

Example

```
Fibonacci Recursion
```

Very slow.

---

# 9. O(N!)

Appears when generating all permutations.

Example

```
Permutations

Backtracking
```

Practically impossible for large `N`.

# Rules to Calculate Time Complexity

## Rule 1: Ignore Constants

```java
for(int i = 0; i < 2 * n; i++){

}
```

```
O(2N)

↓

O(N)
```

---

## Rule 2: Ignore Smaller Terms

```java
O(N² + N + 100)
```

Largest term dominates.

Answer

```
O(N²)
```

---

## Rule 3: Sequential Loops Add

```java
for(...){

}

for(...){

}
```

```
O(N)

+

O(N)

=

O(2N)

=

O(N)
```

---

## Rule 4: Nested Loops Multiply

```java
for(...){

    for(...){

    }

}
```

```
O(N × N)

=

O(N²)
```

---

## Rule 5: Different Variables

```java
for(int i = 0; i < n; i++){

}

for(int j = 0; j < m; j++){

}
```

```
O(N + M)
```

Nested

```java
for(int i = 0; i < n; i++){

    for(int j = 0; j < m; j++){

    }

}
```

```
O(N × M)
```

# Best, Average & Worst Case

Suppose you're searching for a number in an array.

### Best Case

Found at first position.

```
O(1)
```

### Worst Case

Found at last position.

```
O(N)
```

### Average Case

Somewhere in the middle.

```
O(N)
```

When discussing Big-O, we generally refer to the **worst-case**.

# Time Complexity in Competitive Programming

Approximate limits for **1 second**:

| Operations | Usually Acceptable? |
|------------|---------------------|
| 10⁶ | ✅ Yes |
| 10⁷ | ✅ Usually |
| 10⁸ | ⚠ Depends |
| 10⁹ | ❌ Too Slow |

### Choosing an Algorithm Based on Constraints

| N | Recommended Complexity |
|---|------------------------|
| ≤ 20 | O(2ᴺ), O(N!) |
| ≤ 500 | O(N³) |
| ≤ 2,000 | O(N²) |
| ≤ 10⁵ | O(N log N) |
| ≤ 10⁶ | O(N) |
| Very Large | O(log N) or O(1) |

# Common Mistakes

## 1. Counting Every Statement

Wrong

```java
for(int i = 0; i < n; i++){
    a++;
    b++;
    c++;
}
```

Not

```
O(3N)
```

Correct

```
O(N)
```

---

## 2. Ignoring Nested Loops

Wrong

```java
for(...){

    for(...){

    }

}
```

Thinking

```
O(N)
```

Correct

```
O(N²)
```

---

## 3. Confusing Time with Runtime

Time Complexity is about **growth**, not actual execution time.

# Competitive Programming Tips

- Always calculate the time complexity before coding.
- Compare your complexity with the problem constraints.
- Prefer `O(N log N)` over `O(N²)` for large inputs.
- Learn to recognize common patterns:
  - Single loop → `O(N)`
  - Nested loops → `O(N²)`
  - Halving each iteration → `O(log N)`
- Optimizing time complexity is often the key to avoiding **Time Limit Exceeded (TLE)**.

---

# Revision Notes

- Time Complexity measures the growth of operations with input size.
- Big-O represents the worst-case complexity.
- Ignore constants and smaller terms.
- Sequential loops add; nested loops multiply.
- `O(1)` is the fastest, `O(N!)` is the slowest.
- Binary Search runs in `O(log N)`.
- Array traversal is `O(N)`.
- Nested loops are usually `O(N²)`.
- Choose algorithms based on input constraints.
- Always analyze complexity before implementing the solution.
