# Topic: Switch Statements

A **Switch Statement** is a **multi-way selection statement** used to execute one block of code among many based on the value of a single expression.

Instead of writing multiple `if-else if` conditions, a `switch` statement provides a cleaner and more readable way to handle multiple fixed cases.

---

# Why Do We Need Switch?

Suppose you want to print the day of the week based on a number.

Using `if-else`:

```java
if(day == 1){
    System.out.println("Monday");
}
else if(day == 2){
    System.out.println("Tuesday");
}
else if(day == 3){
    System.out.println("Wednesday");
}
...
```

Using `switch`:

```java
switch(day){
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    case 3:
        System.out.println("Wednesday");
        break;
    default:
        System.out.println("Invalid Day");
}
```

The `switch` version is much cleaner and easier to maintain.

---

# Syntax of Switch Statement

```java
switch(expression){

    case value1:
        // Code
        break;

    case value2:
        // Code
        break;

    case value3:
        // Code
        break;

    default:
        // Default Code
}
```

---

# Flow of Execution

```
          Expression
               │
               ▼
     ┌────────────────┐
     │ Compare Cases  │
     └────────────────┘
      │    │     │
      ▼    ▼     ▼
   Case1 Case2 Case3
      │
      ▼
 Execute Block
      │
   break;
      │
      ▼
     End
```

# Example 1: Day of Week

```java
int day = 3;

switch(day){

    case 1:
        System.out.println("Monday");
        break;

    case 2:
        System.out.println("Tuesday");
        break;

    case 3:
        System.out.println("Wednesday");
        break;

    default:
        System.out.println("Invalid Day");
}
```

Output

```
Wednesday
```

---

# Example 2: Calculator

```java
char op = '+';

int a = 10;
int b = 5;

switch(op){

    case '+':
        System.out.println(a + b);
        break;

    case '-':
        System.out.println(a - b);
        break;

    case '*':
        System.out.println(a * b);
        break;

    case '/':
        System.out.println(a / b);
        break;

    default:
        System.out.println("Invalid Operator");
}
```

Output

```
15
```

# The `break` Statement

The `break` statement terminates the current case and exits the switch.

Without `break`, execution continues into the next case.

Example

```java
int x = 2;

switch(x){

    case 1:
        System.out.println("One");

    case 2:
        System.out.println("Two");

    case 3:
        System.out.println("Three");
}
```

Output

```
Two
Three
```

Reason:

After matching `case 2`, Java continues executing subsequent cases because no `break` is present.

---

# Fall-Through Behavior

When `break` is omitted intentionally, multiple cases can execute.

Example

```java
char grade = 'B';

switch(grade){

    case 'A':
    case 'B':
    case 'C':
        System.out.println("Pass");
        break;

    default:
        System.out.println("Fail");
}
```

Output

```
Pass
```

This technique is useful when multiple cases share the same logic.

---

# Default Case

The `default` block executes when no case matches.

Example

```java
int month = 15;

switch(month){

    case 1:
        System.out.println("January");
        break;

    default:
        System.out.println("Invalid Month");
}
```

Output

```
Invalid Month
```

The `default` block is optional but recommended.

# Data Types Supported by Switch (Java)

Java supports:

- `byte`
- `short`
- `char`
- `int`
- `String`
- `enum`

Example

```java
String fruit = "Apple";

switch(fruit){

    case "Apple":
        System.out.println("Red");
        break;

    case "Banana":
        System.out.println("Yellow");
        break;
}
```

---

# Switch Expression (Java 14+)

Modern Java introduced a cleaner switch syntax.

```java
int day = 2;

String result = switch(day){

    case 1 -> "Monday";

    case 2 -> "Tuesday";

    case 3 -> "Wednesday";

    default -> "Invalid";
};

System.out.println(result);
```

Advantages:

- No `break` required.
- No accidental fall-through.
- More concise and readable.

---

# Switch vs If-Else

| Feature | Switch | If-Else |
|----------|---------|----------|
| Checks | One expression | Any condition |
| Operators | Equality only | Any logical/relational operator |
| Readability | Better for many fixed values | Better for complex conditions |
| Performance | Often slightly faster | Good for general logic |
| Range Checks | ❌ | ✅ |

---

# When Should You Use Switch?

Use `switch` when:

- Comparing one variable.
- Checking fixed values.
- Creating menus.
- Selecting commands.
- Mapping numbers to names.

Example

```java
switch(choice){

    case 1:
        startGame();
        break;

    case 2:
        settings();
        break;

    case 3:
        exit();
        break;
}
```

---

# When NOT to Use Switch?

Avoid `switch` when:

- Using ranges.

Example

```java
if(marks >= 90){

}
```

- Using multiple variables.

```java
if(age >= 18 && hasLicense){

}
```

- Using complex logical conditions.

---

# Time Complexity

Switch performs a constant-time selection for a fixed number of cases.

```
Time Complexity : O(1)
Space Complexity : O(1)
```

# Common Mistakes

## 1. Forgetting `break`

Wrong

```java
case 1:
    System.out.println("One");

case 2:
    System.out.println("Two");
```

Output

```
One
Two
```

Correct

```java
case 1:
    System.out.println("One");
    break;
```

---

## 2. Missing Default Case

Wrong

```java
switch(day){

    case 1:
        ...
}
```

Better

```java
default:
    System.out.println("Invalid Input");
```

---

## 3. Using Ranges

Wrong

```java
case > 10:
```

Not allowed.

Use `if-else` instead.

---

## 4. Duplicate Case Values

Wrong

```java
case 1:
case 1:
```

Compilation Error.

---

# Competitive Programming Tips

- Use `switch` for menu-driven programs or fixed command processing.
- Use `if-else` for inequalities (`>`, `<`, `>=`, `<=`).
- Always include `break` unless fall-through is intentional.
- Prefer the modern switch expression (`->`) if using Java 14 or later.

---

# Revision Notes

- `switch` selects one block based on a single expression.
- `case` defines each possible value.
- `break` exits the switch.
- `default` handles unmatched values.
- Omitting `break` causes fall-through.
- `switch` is ideal for fixed choices.
- Use `if-else` for ranges and complex conditions.
- Java supports `String` in switch statements.
- Modern Java offers `switch` expressions with `->`.
